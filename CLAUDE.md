## Git Workflow

- Before starting a new piece of work, always checkout `main`, pull the latest, and create a new feature branch.
- Iterative work on a feature (fixes, refinements) should stay on that feature's branch until it is merged.
- When a PR has been created and merged, the next task must start on a fresh branch from `main` — regardless of which branch is currently checked out.
- At the end of a task, offer to commit, push, and create a PR.
